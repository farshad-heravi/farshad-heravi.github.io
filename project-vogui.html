Autonomous Exploration and Mapping with a Mobile Manipulator

Simulating a Mobile Manipulator (Robotnik Vogui+ & UR5e) for Autonomous Exploration and Mapping

Timeline: Early 2026
Type: Simulation
Role: Robotics Software Developer

Project Overview

The Goal

The goal of this project was to simulate a mobile manipulator consisting of a Robotnik Vogui+ rover equipped with a UR5e robotic arm. The system is designed to autonomously explore an unknown environment and map it using SLAM techniques. The simulation environment, running on ROS 2 Jazzy and Gazebo Harmonic, provides a realistic platform for developing and testing autonomous navigation and manipulation capabilities.

Key Outcomes

  • Successfully integrated a complete simulation of a mobile manipulator using ROS 2 Jazzy and Gazebo Harmonic.
  • Implemented the Navigation2 (Nav2) stack for the rover operating in ackermann mode.
  • Integrated slam_toolbox for online, asynchronous 2D mapping of the unknown environment.
  • Configured explore_lite for frontier-based autonomous exploration to ensure full map coverage.
  • Configured MoveIt 2 for the UR5e robotic arm for future manipulation tasks.
  • Containerized the entire multi-node simulation (Gazebo, Map, Nav, Explore, MoveIt) using Docker Compose for reproducibility and scalable deployment.

Tech Stack

ROS2 Jazzy Gazebo Harmonic Nav2 SLAM MoveIt2 Docker C++ Python

My Role

As the Robotics Software Developer, my responsibilities included designing the simulation architecture, integrating various ROS 2 packages, and ensuring the reliability of the autonomous systems.

  • Simulation Setup: Developed and launched the Gazebo simulation environment including the specific mobile manipulator configurations and custom world environments.
  • Navigation and Mapping: Finely tuned the Nav2 stack for ackermann kinematics and configured SLAM toolbox parameters to achieve high-quality maps. Implemented robust frontier exploration to autonomously drive the robot to unmapped areas.
  • Manipulation: Setup MoveIt 2 for the UR5e arm, validating its configuration alongside the moving base.
  • Deployment: Built a comprehensive Dockerized environment to manage dependencies easily, utilizing docker-compose to coordinate multiple containers (simulation world, robot spawner, SLAM, navigation, exploration, and MoveIt).

Autonomous Exploration Workflow

The autonomous exploration relies heavily on a synergistic approach between `slam_toolbox`, `explore_lite`, and `Nav2`. Nav2 then generates path plans for the rover in ackermann mode to reach these frontiers avoiding newly mapped obstacles dynamically.
